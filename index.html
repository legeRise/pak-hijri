API Quick Start

`latest.json` gives the currently active Pakistan Hijri month start. Your app calculates the day number from that date.

GET data/latest.json
day = diff(today, month_start) + 1

Use the result normally until day 28. On day 29 or 30, hit the endpoint again because the next month may already be published.

Integration Guide

For apps that want to show Pakistan source-backed Hijri dates.

1. Fetch Latest

Request the full `latest.json` URL shown above. It does not return today's Hijri day. It returns the start date of the active Pakistan Hijri month.

{
  "hijri_year": 1448,
  "month": 1,
  "month_name": "Muharram",
  "month_start": "2026-06-17",
  "status": "verified"
}

2. Calculate Daily

Each day, calculate the current Hijri day from `month_start` using the device/app Gregorian date.

day =
  daysBetween(today, month_start) + 1

Example: if `month_start` is `2026-06-17`, then `2026-06-17` is day 1 and `2026-06-18` is day 2.

3. Refetch Near End

Hijri months are 29 or 30 days. Day 29 means the next month might begin tomorrow. Day 30 means the app should check again before trusting the old month.

if day <= 28:
  show date
if day == 29 or day == 30:
  refetch latest.json
if day > 30:
  data is stale

When Data Is Stale

If the calculated day becomes 31 and `latest.json` still points to the old month, the Pakistan dataset has not been updated yet. Your app should stop extending that month.

Year Files

`data/1448.json`, `data/1449.json`, and later year files are the archive. `latest.json` is only the small pointer apps fetch first.

Sources

Each verified month can include one or more sources. A source needs a name, and may also include a description, date, and link.
